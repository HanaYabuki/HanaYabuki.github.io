照例先来 排序算法总览

本文详解 O(nlogn) 的几种排序算法, 代码实现以 Java 为例.

所有的排序均为对整型变量序列int[]排序, 结果均为升序.

归并排序 MergeSort

原理

归并排序是编程中递归思想的绝佳体现.

简单来讲, 归并排序采用分治思想

  1. 将整个数组分成左右两个部分, 分别对左右两个部分排序
  2. 排序以后合并两个数组, 方法是 : 使用两个指针分别指向子序列开始, 一个指针指向待填充序列. 每步比较两指针的值大小, 小的一方填入带填充位置, 并指针后移, 随后待填充序列指针后移. 直到无法比较, 填充剩余序列.

两步骤中, 第一步我们需要已经排好的两份序列, 而他们的排序方式就是这个方式本身. 这很好的体现了递归思想.

实用分析

归并排序的比较都是根据相邻元素的, 相比快速排序来讲, 他是稳定排序.
弊端是必须引入额外空间, 而且在数据逐渐增多时, 元素的操作越来越频繁, 优势逐渐减弱.

希尔排序 ShellSort

原理

希尔排序原理基于插入排序. 由于插入排序对于较为有序的序列来说是高效的, 希尔排序正是利用了这一点来优化其本身的算法.

  1. 取一个小于序列长度的整数 n , 作为当前步长. 把序列的元素分成 n 组, 所有互相之间距离为 n 的元素在这个组内. (即, 每隔n元素就抽出来放到同一组)
  2. 每组内进行插入排序
  3. 取一个更小的增量 n, 重复上述操作, 直到 n=1, 完成最后一次插入排序.

代码实现

实用分析

希尔排序算法的效率不是最高的, 但是他有很深刻的历史意义. 希尔排序从实践的角度, 验证了排序是可以被算法优化的.

现在基本上不再使用希尔排序, 其本身的思路也较为繁琐.

快速排序 QuickSort

原理

快速排序是最普遍使用的排序方案. 很多语言内置的排序算法底层中所使用的(例如C++的sort()), 就是快速排序.

快速排序又叫 划分交换排序, 本质上也使用了分治思想. 但是, 他的思路更加难以理解, 需要详细阐述.

  1. 在整个序列中, 选择末尾元素, 作为一个基准.
  2. 声明两个指针p1,p2, 依次 从左至右和从右至左 遍历整个序列. 当p1元素大于基准时停止之, 当p2元素小于基准时停止之. 此时, 交换p1,p2指向空间的值. 继续遍历, 直到 p1=p2 , 停止折磨.
  3. 考察指针停止的位置, 如果该位置比原基准值大, 则要于基准值交换. 反之, 下一次处理的序列将要改变 : 即基准值变为该指针后面最近的一个值. (这一部分比较难以理解, 目的是为了防止指针停止位置的值没有被考察而产生错误后果.)
  4. 取基准值左右两部分序列, 重复上述操作.

实用分析

快速排序是最实用的算法. 他能够适应几乎任何情况的排序(例如字符串的字典序, 可以通过重写比较函数来实现), 而且时空复杂度都是极底的(不是最低).

缺点在于快排不是稳定的算法, 过多次数的交换操作导致了他的不稳定性.

堆排序 HeapSort

原理

堆排序并不需要完整的构建一个堆, 而是恰当的使用构建堆的原理.

恰当使用位运算

当我们构建一个完全二叉树时, 考察顺序构建时[索引号+1](二进制)的情况, 会出现神奇的效果 :

也就意味着, 基于这一点, 我们可以通过位运算来直接找到每个值的索引位置.

这里使用 [索引-1] 的方式也能获得类似的效果, 自然在实现细节上需要做相应调整

堆的调整

以大顶堆为例, 堆的调整方法为向下调整 :

  1. 如果该节点的叶节点中, 较大者大于他, 则该节点与该叶节点交换
  2. 如果交换了, 则继续向下调整被换掉的叶节点

我们构建堆时, 首先即需要基于原序列构建无序堆, 随即对所有的第一个非叶节点进行向上调整, 即可完成建堆

最终整合

整理知识点, 理清算法 :

  1. 通过序列构建无序堆.
  2. 向下调整, 将无序堆有序化.
  3. 依次从后至前遍历序列, 每次将该节点和堆顶交换, 并完成向下调整, 注意不要换掉已经整体交换过的元素.

实用分析

堆排序不常用, 但是堆这种数据结构十分常见. 可以用于实现优先权队列, 构建哈夫曼树等等. 在操作系统中也有丰富的应用.
