值得注意的是：ref 和 out 并不构成重载。

一、静态 static

细心点的同学应该已经发现了，程序中出现了许多 static 关键字。他是蓝色的，说明他是一个程序保留字。

static ，字面意思：静态的，不变的。在某个类中只有一个，不会因实例化对象的不同而不同。

static 可以修饰类、字段、属性、方法等。如一个方法前加上 static 修饰后要对其进行调用可以直接通过类名点出来，不用再对类进行实例化。

我们仍然使用宝可梦作为例子讲解今天的内容。

1. 静态成员

静态成员比较容易理解。

我们通常声明的 类的成员，声明结果是属于对象的。当我们实例化一个对象时，对应的内存空间被开辟。 而 静态成员 是属于一个类的，只要这个类被定义好了，对应的内存空间就被开辟了。

当然，每个类的每个成员只被定义一次。

不同的是，静态成员可以在被定义的时候初始化，就像普通的局部变量一样。

静态方法很特殊。和静态成员类似， 静态方法属于一个类，而不属于某一个对象。 我们调用的时候可以直接通过类名调用，而不需要实例化一个类。

因为这一关键特性，导致了：

  • 静态方法最多只能访问静态成员，调用静态方法，不能访问非静态成员，调用非静态方法。（因为即使可以访问非静态，程序也并不知道这个非静态成员指的是哪一个实例对象）
  • 静态方法不能用实例调用，只能用类调用。

3. 静态类

静态类与非静态类的重要区别在于静态类没有实例。

静态类的本质是一个抽象的密封类，他密封了许多有共同含义的元素，所以不能被继承，也不能被实例化。

在声明一个类时使用static关键字，具有两个方面的意义：

  • 防止程序员写代码来实例化该静态类
  • 防止在类的内部声明任何非静态字段或方法。

三、更多的动态多态性：接口

抽象类，其实并不是一个面向对象的优雅实现方案，本质上算是一个历史遗留问题。在C++中，由于存在多继承，抽象类可以很好的实现动态多态性。而在C#中，摒弃了难以理解的多继承概念，而使用了接口来代替实现。虽然增加了概念，但是总的来看，这使得C#的程序要比C++更加简单易懂。

说来惭愧，上节课我讲混了虚基类和抽象类，因为抽象类太难用了，老娘从来不用。

接口定义了所有类继承接口时应遵循的”语法合同”。接口定义了语法合同 “是什么” 部分，而 “怎么做” 部分则有接口的实现类决定。

接口中可以声明方法，如果一个类想要实现这个接口，那么这个接口中所有的方法都需要被这个类实现。

五、数据容器

略讲。

有些人喜欢叫这个”数据结构”，我怕吓到大家，换个名字。

数据容器很简单，但是他的底层结构很复杂，计算机专业在大二应该会学到《数据结构》这一科目，会讲到其中很多容器的底层实现。

数据容器是基于泛型实现的，泛型我们会在下节课讲（我不想拖堂了），在深入学习泛型之前，理解数据容器也不难。

C# 中的数据容器有很多很多很多

  • Array 我们最普通的数组，固定长度，只能存储相同的值类型
  • List 加强版数组，不定长，只能储存相同类型（不限制值类型）
  • ArrayList 超级加强版数组。不定长，可以储存乱七八糟的不同的引用类型
  • LinkedList 链表，只能储存相同类型（不限制值类型）
  • Queue 队列（链表实现）
  • Stack 栈（链表实现）
  • Set 集合 > HashSet 哈希集合 SortedSet 排序集合
  • Hashtable 哈希表
  • Dictionary 字典
    不全

数据容器是对整块数据的封装。通常我们用CRUD来概括描述对数据容器的操作

  • C ：create 增加
  • R ：retrieve 检索
  • U ：update 更新
  • D ：delete 删除
