解析JavaWeb三大组件

发表于 2020-12-14 更新于 2020-12-15 分类于 Java基础

本文讲解 Java EE 的三神器

  • 🔪 草薙剑
  • 💎 八尺琼勾玉
  • 🔎 八咫之镜

胡说的, 是 Java EE 中Servlet规范的三大组件

  • Servlet
  • Filter
  • Listener

本文内容主要为后续深入研究 Spring 打基础, 所以不包含任何实例, 只有原理剖析.
后续可能会有三神器在 Spring 中的实现.

Servlet Java服务器程序

简述定义

Java Servlet 是运行在 Web 服务器或应用服务器上的程序，它是作为来自 Web 浏览器或其他 HTTP 客户端的请求和 HTTP 服务器上的数据库或应用程序之间的中间层。

架构示意图

Servlet 主要执行以下任务 :

  • 读取客户端（浏览器）发送的显式的数据。这包括网页上的 HTML 表单，或者也可以是来自 applet 或自定义的 HTTP 客户端程序的表单。
  • 读取客户端（浏览器）发送的隐式的 HTTP 请求数据。这包括 cookies、媒体类型和浏览器能理解的压缩格式等等。
  • 处理数据并生成结果。这个过程可能需要访问数据库，执行 RMI 或 CORBA 调用，调用 Web 服务，或者直接计算得出对应的响应。
  • 发送显式的数据（即文档）到客户端（浏览器）。该文档的格式可以是多种多样的，包括文本文件（HTML 或 XML）、二进制文件（GIF 图像）、Excel 等。
  • 发送隐式的 HTTP 响应到客户端（浏览器）。这包括告诉浏览器或其他客户端被返回的文档类型（例如 HTML），设置 cookies 和缓存参数，以及其他类似的任务。

总结来讲 : 显式或隐式的收发 及 处理 数据

代码分析

Servlet 可以使用 javax.servlet 和 javax.servlet.http 包创建.

常用的实现方式有三种 :

  1. 实现 Servlet 接口
  2. 继承 GenericServlet 类
  3. 继承 HttpServlet 类

我们先不急写, 一层一层观察他的工作结构

Servlet接口

Servlet接口很简单, 就五个函数

init 初始化

传入 ServletConfig对象, 初始化 Servlet容器

service 响应请求

引用传入 Request 和 Response, 适当的时候负责调用 doGet, doPost 方法. 我们需要根据客户端的类型重写.

destroy 摧毁

生命周期只有三个函数, 一目了然. 其余两个都是简单的 get, 不做解释.

观察 GenericServlet 类

可以看到, GenericServlet类同样继承了 Servlet 接口, 这种继承实现的区别是 :

  1. 给出了许多默认实现
  2. 更改了 ServletConfig 的作用域, 改为对外开放.
  3. 仍然没有实现 Service, 所以没用….

分析 HttpServlet

我们直接继承 HttpServlet 发现居然没有报错!!!

也就是说 HttpServlet 的 service 方法已经替我们做了复杂的请求方法判断. 我们唯一要做的就是 重写 doGet 和 doPost 方法

到现在, Servlet 的基本结构我们已经理清了.

Filter 过滤器

简述

顾名思义, 可以动态地拦截请求和响应，以变换或使用包含在请求或响应中的信息。

可以实现以下目的：

  • 在客户端的请求访问后端资源之前，拦截这些请求。
  • 在服务器的响应发送回客户端之前，处理这些响应。

代码分析

过滤器是一个实现了 javax.servlet.Filter 接口的 Java 类.
接口共有三条函数

简单观察生命周期
只有 doFilter 一个核心函数. 也就是说, 我们只需要实现这里的内容即可实现过滤.

Listener 监听器

理解监听器首先要明白Java中如何实现异步回调….

简述

监听器用于监听web应用中某些对象、信息的创建、销毁、增加，修改，删除等动作的发生，然后作出相应的响应处理。
当范围对象的状态发生变化的时候，服务器自动调用监听器对象中的方法。
常用于统计在线人数和在线用户，系统加载时进行信息初始化，统计网站的访问量等等。

代码分析

最常用的监听器实现是 ServletContextListener 接口的类

我们可以看到有默认实现的两个方法.

任何标注为@WebListener，且实现了特定接口的类会被Web服务器自动初始。它会在整个Web应用程序初始化完成后，以及Web应用程序关闭后获得回调通知。

我们可以把初始化数据库连接池等工作放到contextInitialized()回调方法中，把清理资源的工作放到contextDestroyed()回调方法中，因为Web服务器保证在contextInitialized()执行后，才会接受用户的HTTP请求。

常用 Listener 还有 :

  • HttpSessionListener：监听HttpSession的创建和销毁事件；
  • ServletRequestListener：监听ServletRequest请求的创建和销毁事件；
  • ServletRequestAttributeListener：监听ServletRequest请求的属性变化事件（即调用ServletRequest.setAttribute()方法）；
  • ServletContextAttributeListener：监听ServletContext的属性变化事件（即调用ServletContext.setAttribute()方法）；

值得注意的是, Web服务器会对每一个WebApp创建一个与之对应的独一无二的实例 ServletContext, 我们在 Listener 里编写的两个回调方法都是对应该实例的创建和销毁.
